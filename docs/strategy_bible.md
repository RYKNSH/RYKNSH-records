## 1. 理念（Philosophy / WHY）

> **「人間の創造性は、制約と対峙した時にのみ爆発する」**

音楽の歴史が証明している。パンクロックは既存音楽への反逆から生まれた。ヒップホップはギターを買えない貧困という制約から生まれた。ボカロ文化は「歌えない」という制約から生まれた。

**最高の芸術は、最も理不尽な制約の中から生まれる。**

しかし現代のテクノロジーは、制約を「取り除く」方向にばかり進化している。AIが曲を書き、映像を作り、マーケティングを最適化する。全てが楽になる世界で、人間の創造性は萎縮し、凡庸が量産される。

RYKNSH recordsは逆を行く。**AIを「制約を取り除く道具」としてではなく、「立ちはだかる理不尽な壁」として配置する。** その壁と対峙した人間の魂の爆発を、世界に届ける。

これが我々の存在理由だ。

### 理念の検証基準
あらゆる意思決定において、以下を問う：
- ❓ それは人間の創造性を「楽にする」のか、「爆発させる」のか？
- ❓ AIは「便利なアシスタント」になっていないか？「乗り越えるべき壁」であり続けているか？
- ❓ プロセスから「痛み」を取り除こうとしていないか？

## 2. ビジョン（Vision / WHERE）

> **「天才たちの魂の軌跡をオープンソース化し、全人類の感情のポートフォリオを構築する」**

### ビジョンの分解

| 要素 | 意味 | 実装 |
|------|------|------|
| 魂の軌跡 | AIという壁にぶつかり、苦しみ、超えていく人間の記録 | Githubのコミット履歴・PR・Issue |
| オープンソース化 | 密室の制作から、プロセス全公開へ | 公開リポジトリ + コミュニティPF |
| 全人類の感情 | 1ジャンルではなく、全文化圏 | LVMH型ポートフォリオ（数百IP） |
| ポートフォリオ | 投資家が株を分散するように、あらゆる感情を面で取る | 各IPのAIチームの目的関数を多様化 |

### ビジョンの到達指標（10年後）
- 同時並走する独立IP: **1,000+**
- 全世界の月間アクティブファン: **1億人**
- 音楽市場シェアではなく「参加型エンタメ市場」の創造者

## 3. ミッション（Mission / HOW）

> **「大衆に迎合しない『狂気のAIエンジン』で、人間の限界を突破させる」**

### ミッションの3本柱

#### 柱①: Mad Editor — 狂気の壁
AIを「アシスタント」ではなく「ヴィラン（敵）」として設計する。市場最適化を破壊し、アーティストに理不尽な芸術的制約を課す。その制約を超えた時にのみ、パラダイムシフトが生まれる。

→ **実装**: LangGraphの `Mad Editor Agent`。予測可能性スコアが閾値を超えた「安全な」作品を自動Reject。

#### 柱②: Open Process — プロセスの公開
完成品を売るのではない。AIの壁と格闘する人間の魂の軌跡を公開する。ファンと制作者の関係を「消費者↔供給者」から「応援者↔挑戦者」に変える。

→ **実装**: Github上で全制作プロセスを公開。コミュニティPFでファンがIssue投票・PRレビュー参加。

#### 柱③: Zero Marginal Cost Scale — 限界費用ゼロのスケール
アーティスト以外の全業務をAIエージェントが処理する。人間のマネージャー・A&R・マーケターのキャパシティ上限を破壊し、同時に数千のIPを並列プロデュースする。

→ **実装**: LangGraph Orchestrator + Mothership Agent（中央AI）による法務・流通・経理の一括処理。

## 4. 戦略（Strategy / WHAT）

### 4.1 組織戦略 — ホールディングス構造

```
RYKNSH records (Holdings / Mothership)
├── ① Velie          — QA/検証（全社の品質基盤）
├── ② 制作会社        — クリエイティブプロダクション
├── ③ グロース会社     — マーケティング+営業の統合エンジン
├── ④ 広報PR会社      — ブランド保護・信頼管理（独立ガバナンス）
├── ⑤ テックラボ      — AI R&D（全社の武器庫）
├── ⑥ コミュニティPF   — ファン接点UI・課金基盤（全社の顔）
└── ⑦ 音楽レーベル     — 全社能力を結集した最終兵器
```

**設計原則**:
1. 内販×外販の二刀流
2. 循環型エコシステム（制作→グロース→広報→レーベルと価値が循環）
3. Velieファースト（全社の品質ゲート）
4. バックエンド統合 / フロント分断（LVMHアーキテクチャ）

### 4.2 収益戦略 — 内販×外販マトリクス

| 会社 | 外販モデル | 内販役割 |

| ①Velie | QA SaaS ($99-499/月) | 全社CI/CDゲート |
| ②制作社 | クリエイティブ受託 (¥5-100万) | アセット供給 |
| ③グロース | Growth SaaS (¥5-50万/月) | 集客・営業 (CPA連動) |
| ④広報PR | PR リテイナー (¥10-100万/月) | 信頼管理 |

### 4.3 顧客獲得戦略 — 4 Tierモデル

| Tier | チャネル | Phase |
|------|---------|-------|
| 🟢 PLG | 無料Tier→セルフサーブ課金 | Phase 1〜 |
| 🟡 Content | AI生成の技術記事/動画/OSS | Phase 1〜 |

**核心**: 人間の営業は0人。内販の成功 = 外販の最強ケーススタディ。

### 4.4 スケール戦略 — LVMH Swarm

- 各IPに**目的関数を偏らせた専属AIチーム**を配置（バイラル特化/芸術特化/ニッチ特化）
- カニバリゼーション防止：AIの多様性アルゴリズムで「人為的に設計されたエコシステム」を構築
- IP育成階層: L0（オーディション）→ L1（ルーキー）→ L2（メジャー/専属Graph）
- Cross-Maison Collaboration: Mothership Agentが「最も化学反応が期待できるIP」を強制マッチング

### 4.5 グローバル戦略

- **Day 1からグローバル**。現地法人不要（全員AI）
- GitHub = 世界共通プラットフォーム。言語の壁なし
- 法務のグローバル対応のみ本社のAI Legal Agentが処理

### 5.4 意思決定フレームワーク

全ての開発判断は以下の階層で検証する：

```
判断が必要な場面
  ↓
① 理念に反していないか？
   「人間の創造性を楽にしてないか？爆発させているか？」
  ↓
② ビジョンに近づくか？
   「オープンソースか？感情のポートフォリオに貢献するか？」
  ↓
③ ミッションの3本柱のどれに該当するか？
   「Mad Editor / Open Process / Zero Marginal Cost」
  ↓
④ 戦略のどのレイヤーか？
   「組織 / 収益 / 顧客獲得 / スケール / グローバル」
  ↓
⑤ 具体的な戦術は？
   「どのPhase？どのKPI？どの技術スタック？」
```

**例**: 「Velieに自動修正機能（AI Fix）を追加すべきか？」
- ① 理念チェック: バグを自動修正する → 開発者が「楽になる」→ ⚠️ 理念に反する可能性
- 判断: 自動修正ではなく「問題の指摘と、なぜ問題なのかの教育的フィードバック」に留める → ✅ 開発者の成長を促進
